ভূমিকা

PBS Net API হলো একটি RESTful সার্ভিস যা ব্যবহারকারী ব্যবস্থাপনা, নোট তৈরি ও শেয়ারিং, এবং অনুমতি নিয়ন্ত্রণের জন্য ডিজাইন করা হয়েছে। সকল অনুরোধ এবং প্রতিক্রিয়া (Request & Response) JSON ফরম্যাটে হবে।

Base URL: https://pbsnet.onrender.com

প্রমাণীকরণ (Authentication)

এই এপিআই-তে JWT (JSON Web Token) ব্যবহার করে প্রমাণীকরণ করা হয়। ব্যবহারকারীকে প্রথমে /auth/login এন্ডপয়েন্টে লগইন করে একটি টোকেন সংগ্রহ করতে হবে। এরপর সুরক্ষিত (Private) এন্ডপয়েন্টগুলো অ্যাক্সেস করার জন্য প্রতিটি অনুরোধের Headers-এ এই টোকেনটি পাঠাতে হবে।

হেডার ফরম্যাট:

Authorization: Bearer <YOUR_JWT_TOKEN>

ব্যবহারকারী রেজিস্টার করা

POST /auth/register

নতুন ব্যবহারকারী তৈরি করার জন্য এই এন্ডপয়েন্ট ব্যবহার করা হয়।

Request Body:

{
    "email": "user@example.com",
    "mobile": "01700000000",
    "password": "strongpassword123"
}

Success Response (201 Created):

{
    "message": "User registered successfully.",
    "user": {
        "id": "c3e4a5b6-...",
        "email": "user@example.com",
        "mobile": "01700000000"
    }
}

ব্যবহারকারী লগইন

POST /auth/login

ইমেইল বা মোবাইল নম্বর এবং পাসওয়ার্ড ব্যবহার করে লগইন করে JWT টোকেন সংগ্রহ করা।

Request Body:

{
    "identifier": "user@example.com",
    "password": "strongpassword123"
}

Success Response (200 OK):

{
    "message": "Login successful.",
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
        "id": "c3e4a5b6-...",
        "email": "user@example.com"
    }
}

পাসওয়ার্ড রিসেট অনুরোধ

POST /auth/forgot-password

পাসওয়ার্ড রিসেট করার জন্য ইমেইলে একটি লিঙ্ক পাঠায়।

Request Body:

{
    "email": "user@example.com"
}

Success Response (200 OK):

{
    "message": "If an account with that email exists, a password reset link has been sent."
}

পাসওয়ার্ড পরিবর্তন করা

PUT /users/password

লগইন করা অবস্থায় বর্তমান পাসওয়ার্ড দিয়ে নতুন পাসওয়ার্ড সেট করা। (Access: Private)

Request Body:

{
    "currentPassword": "oldstrongpassword",
    "newPassword": "newverystrongpassword"
}

Success Response (200 OK):

{
    "message": "Password changed successfully."
}

প্রোফাইল দেখা ও আপডেট করা

GET /users/profile

লগইন করা ব্যবহারকারীর প্রোফাইল তথ্য দেখা। (Access: Private)

Success Response (200 OK):

{
    "id": "c3e4a5b6-...",
    "email": "user@example.com",
    "mobile": "01700000000",
    "name": "Sabbir Ahmed",
    "position": "AGM (IT)",
    "pbs_name": "Dhaka PBS-1"
}

PUT /users/profile

ব্যবহারকারীর প্রোফাইল তথ্য আপডেট করা। (Access: Private)

Request Body:

{
    "name": "Sabbir Ahmed Khan",
    "position": "Sr. AGM (IT)"
}

Success Response (200 OK):

{
    "message": "Profile updated successfully.",
    "user": {
        "name": "Sabbir Ahmed Khan",
        "position": "Sr. AGM (IT)"
    }
}

ব্যবহারকারীকে খোঁজা

GET /users/search?mobile={mobile_number}

GET /users/{userId}

মোবাইল নম্বর অথবা User ID ব্যবহার করে অন্য ব্যবহারকারীকে খুঁজে বের করা। (Access: Private)

Success Response (200 OK):

{
    "id": "d4f5a6b7-...",
    "email": "anotheruser@example.com",
    "mobile": "01800000000",
    "name": "Another User"
}

নতুন নোট তৈরি করা

POST /notes

একটি নতুন নোট তৈরি করা। যিনি তৈরি করবেন, তিনি স্বয়ংক্রিয়ভাবে সেই নোটের Admin হবেন। (Access: Private)

Request Body:

{
    "name": "Monthly Maintenance Report - August",
    "office_name": "Savar Zonal Office",
    "pbs_name": "Dhaka PBS-1",
    "type": "Report",
    "note_data": {
        "status": "Completed"
    }
}

Success Response (201 Created):

{
    "message": "Note created successfully.",
    "note": {
        "id": "A1B2C3D4",
        "name": "Monthly Maintenance Report - August"
    }
}

নোটের বিবরণ দেখা

GET /notes/{noteId}

একটি নির্দিষ্ট নোটের সম্পূর্ণ বিবরণ দেখা। (Access: Admin/Viewer of the note)

Success Response (200 OK):

{
    "id": "A1B2C3D4",
    "name": "Monthly Maintenance Report - August",
    "note_data": {
        "status": "Completed"
    }
}

নোট আপডেট করা

PUT /notes/{noteId}

একটি নির্দিষ্ট নোটের তথ্য পরিবর্তন করা। (Access: Admin of the note)

Request Body:

{
    "name": "Monthly Report - August (Revised)",
    "note_data": {
        "status": "Approved"
    }
}

Success Response (200 OK):

{
    "message": "Note updated successfully.",
    "note": {
        "id": "A1B2C3D4",
        "name": "Monthly Report - August (Revised)"
    }
}

নোট ডিলিট করা

DELETE /notes/{noteId}

একটি নোট স্থায়ীভাবে মুছে ফেলা। (Access: Admin of the note)

Success Response (200 OK):

{
    "message": "Note deleted successfully."
}

নোটে ব্যবহারকারী যুক্ত করা

POST /notes/{noteId}/permissions

অন্য ব্যবহারকারীকে নোটে Admin বা Viewer হিসেবে যুক্ত করা। (Access: Admin of the note)

Request Body:

{
    "mobile": "01900000000",
    "role": "viewer"
}

Success Response (201 Created):

{
    "message": "User successfully added as viewer."
}

নোট থেকে ব্যবহারকারী বাদ দেওয়া

DELETE /notes/{noteId}/permissions

নোট থেকে কোনো Admin বা Viewer-কে বাদ দেওয়া। (Access: Admin of the note)

Request Body:

{
    "userIdToRemove": "d4f5a6b7-..."
}

Success Response (200 OK):

{
    "message": "User removed from the note successfully."
}

নোটের সকল ব্যবহারকারী দেখা

GET /notes/{noteId}/users

একটি নির্দিষ্ট নোটের সাথে যুক্ত সকল ব্যবহারকারী (Admin এবং Viewer) এবং তাদের বিস্তারিত তথ্যের তালিকা দেখা। (Access: Admin/Viewer of the note)

Success Response (200 OK):

[
    {
        "role": "admin",
        "id": "c3e4a5b6-...",
        "name": "Sabbir Ahmed",
        "email": "user@example.com",
        "mobile": "01700000000",
        "position": "AGM (IT)",
        "office_name": "Savar Zonal Office",
        "pbs_name": "Dhaka PBS-1",
        "profile_pic_url": null
    },
    {
        "role": "viewer",
        "id": "d4f5a6b7-...",
        "name": "Another User",
        "email": "anotheruser@example.com",
        "mobile": "01800000000",
        "position": "Jr. Engineer",
        "office_name": "Main Office",
        "pbs_name": "Dhaka PBS-1",
        "profile_pic_url": null
    }
]

Error Response (404 Not Found):

যদি নোটটি খুঁজে না পাওয়া যায় অথবা ব্যবহারকারীর অনুমতি না থাকে।

{
    "error": "Note not found or you do not have permission."
}

আমার নোটের তালিকা

GET /dashboard/my-notes

ব্যবহারকারী যে সমস্ত নোটের Admin বা Viewer, সেগুলোর তালিকা দেখা। (Access: Private)

Success Response (200 OK):

[
    {
        "role": "admin",
        "notes": {
            "id": "A1B2C3D4",
            "name": "Monthly Maintenance Report"
        }
    }
]

PBS-এর সব নোট

GET /dashboard/pbs-notes

ব্যবহারকারীর প্রোফাইলের PBS Name-এর অধীনে থাকা সমস্ত নোটের তালিকা দেখা। (Access: Private)

Success Response (200 OK):

[
    {
        "id": "A1B2C3D4",
        "name": "Monthly Maintenance Report",
        "note_data": { "status": "Completed" }
    },
    {
        "id": "K5L6M7N8",
        "name": "Billing Cycle Analysis"
    }
]

দ্রষ্টব্য: যে নোটগুলোতে ব্যবহারকারীর অনুমতি নেই, সেগুলোর `note_data` দেখানো হবে না।

ভিউয়ার হওয়ার অনুরোধ

POST /notes/{noteId}/request-access

কোনো নোটে Viewer হওয়ার জন্য তার Admin-দের কাছে অনুরোধ পাঠানো। (Access: Private)

Success Response (201 Created):

{
    "message": "Viewer access request sent successfully."
}

পেন্ডিং রিকোয়েস্ট দেখা

GET /notes/{noteId}/requests

একটি নির্দিষ্ট নোটের জন্য আসা সমস্ত পেন্ডিং ভিউয়ার রিকোয়েস্টের তালিকা দেখা। (Access: Admin of the note)

Success Response (200 OK):

[
    {
        "id": 1,
        "status": "pending",
        "requester": {
            "id": "e5f6a7b8-...",
            "name": "Requesting User"
        }
    }
]

রিকোয়েস্ট পরিচালনা

PUT /notes/requests/{requestId}

একটি ভিউয়ার রিকোয়েস্ট Accept (গ্রহণ) বা Reject (প্রত্যাখ্যান) করা। (Access: Admin of the note)

Request Body:

{
    "action": "accept"
}

Success Response (200 OK):

{
    "message": "Request accepted. User is now a viewer."
}
